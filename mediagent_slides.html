The Solution
02 — Solution
One form. Multiple calls.
Best slot. Booked.
"I'm calling on behalf of a patient to check availability and insurance acceptance…"
📋
Patient fills one form: name, address, insurance, specialization, date range, health brief
🗺️
Agent finds all nearby matching clinics via Google Maps within specified radius
📞
Makes real outbound calls to each clinic using Twilio Voice, powered by Qwen 14B LLM
🧠
Extracts: insurance accepted?, earliest date, provider name, booking requirements
🏆
Ranks all options, selects earliest valid slot, places final booking call
💬
Sends WhatsApp confirmation with all appointment details to patient instantly

Trust & Safety
05 — Security & Trust
Patient data handled
with guardrails.
🛡️
NeMo Guardrails
NVIDIA NeMo Guardrails design — input/output rails prevent PII leakage and off-topic responses during calls
🔒
No Persistent PII
Patient data is session-scoped. Health briefs used only when clinics ask. No post-session data retention by design
📜
HIPAA-Aligned Design
Architecture intent is HIPAA-aligned. Insurance details, names, phone numbers handled via env-var secrets, never hardcoded
🤐
AI Identity Policy
Agent does not proactively reveal it's AI. Focuses on scheduling logistics only. Uses health brief minimally and only when asked
🧱
Local LLM
Qwen 14B runs fully on-device. No patient data leaves to external LLM APIs. Full data sovereignty
📊
Full Audit Log
Every call attempt, transcript turn, extracted fact, and booking result is logged in SQLite for full transparency
